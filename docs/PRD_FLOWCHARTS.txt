## 1. Authentication — Consumer

```mermaid
flowchart TD
    A([Consumer opens app]) --> B{Has account?}
    B -- No --> C[Enter phone + username + displayName + citySlug]
    C --> D[Consent to analytics opt-in]
    D --> E[POST /auth/consumer/signup]
    E --> F[Check phone uniqueness]
    F --> G[Validate citySlug]
    G --> H[OTP rate limit check\n3/hr · 60s cooldown]
    H --> I[Create Cognito user]
    I --> J[Create DB user record]
    J --> K[Store consent record]
    K --> L[Initiate Cognito auth → OTP sent to phone]
    L --> M[User enters OTP]
    M --> N[POST /auth/consumer/verify-otp]
    N --> O[Validate OTP via Cognito challenge]
    O --> P[Return accessToken + refreshToken + user profile]
    P --> Q([Logged in — tier: local])

    B -- Yes --> R[Enter phone number]
    R --> S[POST /auth/consumer/login]
    S --> T[OTP rate limit check]
    T --> U[Initiate Cognito auth → OTP sent]
    U --> M
```

## 4. Staff Invite Flow

```mermaid
flowchart TD
    A([Business owner]) --> B[POST /business/staff/invite with phone or email]
    B --> C{Tier staff limit reached?}
    C -- Yes --> D([Error: Staff limit reached])
    C -- No --> E[Create StaffInvite record with token + expiry]
    E --> F[Invite token sent to staff via SMS/email]
    F --> G([Staff receives invite link])
    G --> H[POST /auth/accept-invite with token + name + phone]
    H --> I{Token valid and not expired?}
    I -- No --> J([Error: Invite expired or invalid])
    I -- Yes --> K[Mark invite as accepted]
    K --> L[Create Cognito staff user]
    L --> M[Create StaffAccount in DB]
    M --> N([Staff account active])
```

## 5. Check-In Pipeline

```mermaid
flowchart TD
    A([User at a venue]) --> B{Check-in method?}
    B -- GPS --> C[Submit lat + lng + nodeId + type]
    B -- QR scan --> D[Submit qrToken + nodeId + type]

    C --> E[Verify GPS proximity within 200m]
    D --> F[Validate QR HMAC token\n15-min rotating window]

    E --> G{Within range?}
    G -- No --> H([Error: Too far from venue])
    G -- Yes --> I

    F --> I[Abuse checks\nfingerprint · rate limit · velocity]
    I --> J{Cooldown active?}
    J -- Yes · reward 4hr --> K([Error: Cooldown active])
    J -- Yes · presence 1hr --> K
    J -- No --> L[Insert CheckIn record]
    L --> M[Increment user totalCheckIns]
    M --> N[Update streak]
    N --> O[Set Redis cooldown key]
    O --> P[Increment nodesPulse score in Redis\nScore = daily x5 + unique x2]
    P --> Q[Increment city leaderboard in Redis]
    Q --> R[Emit pulse update via WebSocket]
    R --> S[Emit anonymous city toast via WebSocket]
    S --> T[Emit friend toasts to mutual followers]
    T --> U{Business owned node?}
    U -- Yes --> V[Emit business check-in event]
    U -- No --> W
    V --> W{type = reward?}
    W -- Yes --> X[Publish to SQS reward queue]
    X --> Y([Reward evaluator worker picks up job])
    W -- No --> Z([Check-in complete])
```

## 6. Reward System

```mermaid
flowchart TD
    A([Business creates reward]) --> B[POST /rewards with nodeId + type + title + slots]
    B --> C{Active reward limit for tier?}
    C -- Exceeded --> D([Error: Reward limit reached])
    C -- OK --> E[Create Reward record in DB]

    F([SQS reward evaluator worker]) --> G[Pull check-in job from queue]
    G --> H[Load reward rules for node]
    H --> I{Trigger condition met?}
    I -- No --> J([No reward issued])
    I -- Yes --> K{Slots available?}
    K -- No --> L([Slots exhausted])
    K -- Yes --> M[Create RewardRedemption record\nwith 6-char redemption code]
    M --> N[Increment claimedCount on Reward]
    N --> O[Notify user via socket or push notification]

    P([User views unclaimed rewards]) --> Q[GET /rewards/unclaimed]
    Q --> R[Show redemption code + expiry]
    R --> S([User at venue shows code to staff])
    S --> T[Staff scans or enters code]
    T --> U[POST /rewards/redeem with code]
    U --> V{Code valid and not expired?}
    V -- No --> W([Error: Invalid or expired code])
    V -- Yes --> X{Staff belongs to owning business?}
    X -- No --> Y([Error: Forbidden])
    X -- Yes --> Z[Mark RewardRedemption as redeemed]
    Z --> AA([Redemption confirmed])
```

## 7. Nodes (Venues)

```mermaid
flowchart TD
    A([Business creates a node]) --> B[POST /nodes with name + category + lat + lng + citySlug]
    B --> C[Auto-generate slug]
    C --> D[Save Node with claimStatus: unclaimed]

    E([Business claims existing node]) --> F[POST /nodes/:id/claim with registrationNumber]
    F --> G{Node already claimed or pending?}
    G -- Yes --> H([Error: Already claimed])
    G -- No --> I[CIPC business registration verification]
    I --> J{CIPC match?}
    J -- Validated --> K[Set claimStatus: claimed · link businessId]
    J -- Unavailable --> L[Set claimStatus: cipc_unavailable]
    K --> M([Node owned by business])

    N([Business updates node]) --> O[PATCH /nodes/:id with name · category · colour · icon · qrEnabled]
    O --> P{Owns node?}
    P -- No --> Q([Error: Forbidden])
    P -- Yes --> R[Update Node record]

    S([User reports a node]) --> T[POST /nodes/:id/report with type + detail]
    T --> U{Reporter suspended? 3 dismissed in 30 days}
    U -- Yes --> V([Error: Reporting privileges suspended])
    U -- No --> W[Create Report record]
    W --> X{type = fake_rewards AND 5+ reports in 24h?}
    X -- Yes --> Y[Auto-flag node]
    X -- No --> Z([Report submitted])
```

## 8. Social — Follow and Activity Feed

```mermaid
flowchart TD
    A([User searches for another user]) --> B[GET /social/users/search?q=query]

    C --> D([User taps Follow])

    I -- No --> K([One-way follow])

    L([User opens Feed tab]) --> M[GET /social/feed]
    M --> N[Load check-ins from mutual follows only]

    Q([User opens Leaderboard tab]) --> R[GET /social/leaderboard/:citySlug]
    R --> S[Fetch top 50 from Redis sorted set]
    S --> T[Resolve profiles from DB]
    T --> U[Apply friend visibility rules]
    U --> V{Is friend or self?}
    V -- Yes --> W[Show displayName + avatar + tier]
    V -- No --> X[Null out identity — show tier only]
    W --> Y([Leaderboard displayed])
    X --> Y
```

## 14. Background Workers

```mermaid
flowchart TD
    A([Pulse Decay Worker — runs hourly]) --> B[For each city node with pulseScore above 0]
    B --> C[Apply decay factor to Redis pulse score]
    C --> D[Emit updated node state via WebSocket]

    E([Node State Evaluator — runs every 15 min]) --> F[Read pulse scores from Redis]
    F --> G{Score range?}
    G -- 61 and above --> H([State: popping])
    G -- 31 to 60 --> I([State: buzzing])
    G -- 11 to 30 --> J([State: active])
    G -- 1 to 10 --> K([State: quiet])
    G -- 0 --> L([State: dormant])

    M([Reward Evaluator Worker — SQS consumer]) --> N[Receive check-in job from SQS]
    N --> O[Load reward rules for node]
    O --> P{Trigger condition met?}
    P -- Yes --> Q[Issue RewardRedemption + notify user]
    P -- No --> R([No reward issued])

    S([Leaderboard Reset Worker — weekly cron]) --> T[Snapshot leaderboard to history table]
    T --> U[Clear live Redis leaderboard sorted set]

    V([Cleanup Worker — daily]) --> W[Expire soft-deleted records]
    W --> X[Process pending POPIA erasure requests]

    Y([Partition Manager — monthly]) --> Z[Create new check_ins table partition for next month]
```
